# 工程核心模块和功能点分析

## 1. 整体架构
- 基于 FastAPI 构建的知识图谱服务
- 采用分层架构：API层、服务层、数据层
- 支持异步操作和数据库交互

## 2. 核心模块

### 2.1 API层 (`kg/api/`)
- **AutoKGRouter**: 自动知识图谱构建路由
  - `/autokg/extract-entities`: 从文本中抽取实体
  - `/autokg/extract-relations`: 从文本中抽取关系
  - `/autokg/process-text`: 处理文本，抽取实体和关系
  - `/autokg/process-news`: 处理新闻，构建知识图谱
  - `/autokg/bulk-process`: 批量处理文本，构建知识图谱

### 2.2 服务层 (`kg/services/`)
- **AutoKGService**: 整合大模型调用和实体关系提取
  - 实体提取
  - 关系提取
  - 三元组提取
  - 文本处理

- **NewsProcessingService**: 新闻处理服务
  - 新闻存储
  - 实体、关系提取
  - 摘要生成

### 2.3 数据层 (`kg/database/`)
- **Entity**: 实体表模型
- **Relation**: 关系表模型
- **News**: 新闻表模型

### 2.4 核心配置 (`kg/core/`)
- 应用配置
- CORS配置
- 数据库配置

## 3. 依赖组件
- LLM服务：大模型调用
- 数据库服务：SQLAlchemy
- 嵌入服务：ThirdPartyEmbeddingService

## 4. 核心功能点
1. 文本实体提取
2. 文本关系提取
3. 文本三元组提取
4. 新闻处理与存储
5. 批量文本处理
6. 知识图谱构建
